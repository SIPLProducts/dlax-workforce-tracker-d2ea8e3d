## Goal
Auto-IDs (`DEP-###`, `CAT-###`) are already generated by DB triggers and shown in Project Assignments. Now surface them in the Departments and Categories master screens so users can see the auto-generated code for each row.

## Changes

### `src/routes/masters.departments.tsx`
- Add a new `Code` column (~110px) as the first column in the table header.
- Render `d.department_code` in each row (muted/monospaced badge style, fallback `—`).
- In the inline add/edit form row, show a placeholder cell ("Auto" for new rows, the existing code when editing) — read-only, no input.
- Update the empty-state `colSpan` from 3 to 4.

### `src/routes/masters.categories.tsx`
- Add a `Code` column as the first column in the table header.
- Render `d.category_code` in each row.
- Update the empty-state `colSpan` from 4 to 5.
- No changes to the create/edit dialog (codes are system-assigned).

## Out of scope
- Editing or regenerating codes.
- Changing prefix format or sequence.
- Any schema or trigger changes (already in place).
